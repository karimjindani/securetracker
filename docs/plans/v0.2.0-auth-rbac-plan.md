# v0.2.0 Auth, Organizations, Users, RBAC Plan

## Summary

Implement Keycloak-backed authentication, local user/organization synchronization, backend RBAC guards, role-aware frontend navigation, and the first protected API surfaces.

## Scope

- Import a local `securetracker` Keycloak realm for development.
- Add demo users for each role in local Keycloak only.
- Validate JWTs in the NestJS API using issuer and JWKS.
- Add `/me`, `/organizations`, and `/users` APIs.
- Create audit records for organization and user writes.
- Add a frontend auth shell with login/logout, session loading, dashboard, organizations, users, and access denied routes.

## Out of Scope

- Application inventory.
- Engagement workflow implementation.
- Report upload.
- Production identity-provider configuration.

## Quality Gates

```text
npm run db:generate
npm run typecheck
npm run lint
npm test
npm run build
docker compose config --quiet
```

## Acceptance Criteria

- Users can authenticate through Keycloak.
- `/me` maps token claims to a local user profile.
- Backend role guards protect organizations and users APIs.
- Frontend navigation changes by role.
- Documentation, testing evidence, and version baseline are updated.
