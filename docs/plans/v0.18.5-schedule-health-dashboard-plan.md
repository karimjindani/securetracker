# v0.18.5 Schedule Health Dashboard Plan

## Summary

Finish schedule-health visibility for VAPT engagements. Dashboard highlights on-track, attention, and at-risk work, and links directly into filtered Engagements Kanban views.

## Key Changes

- Derive engagement schedule health dynamically from planned start/end dates and lifecycle status.
- Add `scheduleHealth` filtering to the Engagements API and Kanban page.
- Add Dashboard schedule-health counts and short attention/risk lists.
- Show colored schedule-health chips on Kanban cards.
- Keep warning threshold at 7 days and exclude terminal statuses from risk counts.

## Test Plan

- Verify shared schedule-health classification for green, yellow, red, and terminal states.
- Verify invalid `scheduleHealth` query values are rejected.
- Verify Engagements API can filter by derived schedule health.
- Verify Dashboard exposes schedule-health counts and lists.
- Verify Dashboard links open Kanban with the expected health filter.

## Assumptions

- Schedule health is derived and not persisted.
- `YELLOW` means needs attention.
- `RED` means at risk.
- System settings for the 7-day threshold remain future Epic 16 work.
