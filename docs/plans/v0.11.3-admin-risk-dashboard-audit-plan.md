# v0.11.3 Admin Pages, Risk Acceptance, Dashboards, and Audit Search Plan

## Summary

Implement the next allocated version under `AGENTS.md` rules. Remote inspection found only `origin/main`, with `v0.11.2` as the highest allocated version, so this slice uses `v0.11.3`.

## Key Changes

- Complete Organizations and Users frontend pages with API-backed list, create, update, loading, empty, and error states.
- Add risk acceptance request and NBP review workflow for unresolved findings.
- Replace static dashboard content with live engagement, finding, risk, revalidation, heatmap, calendar, and vendor metrics.
- Add audit search and CSV export for governance users.
- Keep notifications, reference-data administration, and system configuration for later slices.

## Test Scenarios

- System Admin sees populated Organizations and Users pages and can create/update records.
- Non-System roles cannot write user or organization data; Users navigation is not shown to them.
- Paysys requests risk acceptance for unresolved findings.
- NBP approves and rejects risk acceptance requests.
- Approved risk acceptance moves the linked finding to `RISK_ACCEPTED`.
- Dashboard metrics reflect accepted risks, open findings, and revalidation outcomes.
- Auditor searches audit logs and exports CSV.

## Assumptions

- `AGENTS.md` version allocation takes precedence over the older sequential roadmap label.
- The local demo admin username remains `system.admin`; `super.admin` is treated as a user-facing nickname unless a future request requires renaming Keycloak seed data.
- Notifications are out of scope for this slice.
