# v0.18.8 Production Hardening Plan

## Summary

Prepare SecureTracker for a Docker VM production pilot by adding production Compose assets, startup environment validation, CI security gates, and backup/restore documentation.

## Key Changes

- Add `docker-compose.prod.yml` and `.env.production.example` for a production-like Docker VM deployment without dev Keycloak or Mailpit.
- Add explicit production environment validation through `SECURETRACKER_DEPLOYMENT_MODE=production`.
- Keep local Docker behavior unchanged; local Compose continues to use the development Keycloak and Mailpit stack.
- Expand CI to validate Docker Compose files, build backend/frontend images, run dependency audit, scan for secrets, and run a non-blocking Trivy container scan.
- Add production operations guidance for external OIDC, real SMTP, reverse proxy/TLS, persistent volumes, and backup/restore.

## Test Plan

- Run typecheck, lint, unit tests, build, local Compose config, production Compose config, Docker image builds, dependency audit, regression, cleanup, and seeded reset.
- Confirm backend health reports `0.18.8`.
- Confirm production env validation fails clearly when required production values are missing and does not run for local/dev mode.

## Assumptions

- Production pilot target is a Docker VM, not Kubernetes.
- External NBP/client OIDC and production SMTP are supplied by deployment operations.
- Seeded reset is local/dev only and must not be used against production data.
