# v0.3.1 Plan: Regression Suite and Ops Console

## Summary

Add a local/dev Ops Console and executable regression suite for SecureTracker. The suite validates platform health, authentication, RBAC, applications, calendar, and protected operational maintenance workflows.

## Key Changes

- Add Playwright regression tests under `tests/e2e`.
- Add support helpers for seeded demo users, Keycloak token acquisition, API clients, and regression data factories.
- Add scripts for e2e/regression execution, regression data cleanup, and reset to seeded baseline.
- Add external host-run Ops Console for health, container status, regression runs, cleanup, and reset.
- Keep Ops outside the SecureTracker frontend/backend containers.
- Bind Ops to `127.0.0.1:3300` with optional `OPS_CONSOLE_TOKEN`.

## Test Plan

- Run the standard SDLC gates.
- Run `npm run test:e2e` and `npm run test:regression`.
- Run `npm run test:data:cleanup`.
- Run `npm run reset:seeded`.
- Verify the external Ops Console reports Dockerized app health.
- Verify regression, cleanup, and reset run from the host and stream logs.

## Assumptions

- Ops Console is a local/dev capability only.
- Regression data is identified by `REGRESSION_`.
- Browser smoke tests are enabled with `RUN_BROWSER_E2E=true`; API-backed Playwright regression runs by default.
