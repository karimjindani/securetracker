# v0.1.0 Implementation Plan

## Summary

Create the SecureTracker foundation as a Dockerized monorepo with React/Vite frontend, NestJS backend, Prisma/PostgreSQL data model, shared workflow constants, CI gates, and SDLC documentation scaffolding.

## Scope

- Add monorepo workspaces for `apps/frontend`, `apps/backend`, and `packages/shared`.
- Add Prisma schema for the cleaned VAPT workflow.
- Add Docker Compose services for PostgreSQL, Keycloak, MinIO, SMTP test service, and backend API.
- Add root Dockerfile and GitHub Actions CI.
- Add backend health endpoint and frontend landing shell.
- Add version baseline, deployment, CI/CD, source-control, user-guide, and testing docs.
- Remove remaining obsolete initial-scope governance contradictions before the code baseline.

## Quality Gates

Run the required SDLC sequence:

```text
npm run db:generate
npm run typecheck
npm run lint
npm test
npm run build
```

## Security Notes

- PDF passwords, test account passwords, tokens, and secrets must never be stored or logged.
- Business records use auditable updates and avoid hard deletes.
- RBAC rules begin in shared constants and will be enforced in API guards in `v0.2.0`.

## Acceptance Criteria

- The repository has a working application scaffold.
- Docker Compose can start core dependency services.
- CI runs database generation, typecheck, lint, tests, and build.
- Docs are updated for this version baseline.
