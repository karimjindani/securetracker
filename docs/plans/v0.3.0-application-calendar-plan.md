# v0.3.0 Plan: Application Inventory and Annual VAPT Calendar

## Summary

Implement the first business workflow slice after authentication: application inventory and planned VAPT calendar entries. Calendar entries are stored as VAPT engagements in `PLANNED` status; full lifecycle transitions remain reserved for `v0.4.0`.

## Key Changes

- Add authenticated application inventory APIs and UI for create, view, search, and update.
- Add VAPT calendar APIs and UI for annual and ad-hoc planned engagements.
- Enforce RBAC:
  - `SYSTEM_ADMIN` and `PAYSYS_SECURITY_ADMIN` can manage applications.
  - `SYSTEM_ADMIN`, `NBP_SECURITY_ADMIN`, and `PAYSYS_SECURITY_ADMIN` can manage calendar entries.
  - Other permitted roles can view applications and calendar entries.
- Store calendar entries as `VaptEngagement` records with status `PLANNED`.
- Audit application and calendar create/update actions.
- Preserve the documented assessment types: `WHITEBOX` and `BLACK_GREY`.

## Test Plan

- Run SDLC gates:
  - `npm run db:generate`
  - `npm run typecheck`
  - `npm run lint`
  - `npm test`
  - `npm run build`
  - `docker compose config --quiet`
- Verify backend service tests for application creation, calendar creation, validation, and audit logging.
- Verify shared permission/navigation tests cover v0.3 application and calendar access.
- Manually verify authenticated users can see the new navigation and role-restricted create forms.

## Assumptions

- `v0.3.0` starts from the `v0.2.0` auth/RBAC baseline.
- No schema migration is required because `Application` and `VaptEngagement` already exist in Prisma.
- Engagement initiation, scoping, reports, findings, and lifecycle transitions are out of scope until later versions.
