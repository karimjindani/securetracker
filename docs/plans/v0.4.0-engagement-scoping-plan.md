# v0.4.0 Plan: Engagement Initiation, Scoping, and Lifecycle Workflow

## Summary

Implement engagement management on top of planned VAPT calendar entries. Add Paysys-Apprise initiation, scoping records, workflow transitions, seeded test records, and regression coverage.

Bank/NBP remains optional for the first Paysys-Apprise initiation meeting. NBP does not approve initial scope. NBP Security Admin remains the only role authorized to mark an engagement `Closed`.

## Key Changes

- Add backend engagement APIs for list, detail, metadata update, and status transitions.
- Add backend scoping APIs for create, update, list, and finalize.
- Add frontend `/engagements` and `/engagements/:id` pages.
- Add `ScopingRecordStatus` values `DRAFT` and `FINAL`.
- Add seeded v0.4.0 applications, engagements, and scoping records for repeatable regression.
- Audit engagement updates, status changes, scoping creation, scoping updates, and scoping finalization.

## Seeded Records

Reset and seed restore:

- Applications:
  - `Seeded Core Banking Portal`
  - `Seeded Mobile Banking API`
  - `Seeded Internet Banking Web`
- Engagements:
  - `Seeded Core Banking Planned Whitebox VAPT` in `PLANNED`
  - `Seeded Mobile Banking Paysys-Apprise Initiated VAPT` in `PAYSYS_APPRISE_INITIATED`
  - `Seeded Internet Banking Apprise Assessment VAPT` in `APPRISE_ASSESSMENT`
  - `Seeded Core Banking NBP Closing Meeting VAPT` in `NBP_IS_REVIEW_CLOSING_MEETING`
  - `Seeded Mobile Banking Closed VAPT` in `CLOSED`
- Scoping records:
  - Draft scoping record with Paysys + Apprise participants only.
  - Final scoping records with optional NBP participant included.

No seeded record contains real credentials or passwords.

## Test Plan

- Run SDLC gates: `db:generate`, `typecheck`, `lint`, `test`, `build`, and `docker compose config --quiet`.
- Run regression gates: `test:e2e`, `test:regression`, `test:data:cleanup`, and `reset:seeded`.
- Verify Paysys can create/finalize scoping without NBP approval.
- Verify Bank/NBP optional attendance is accepted.
- Verify non-NBP roles cannot close engagements.
- Verify NBP can close and Paysys can move closed engagements to `GO_LIVE`.
