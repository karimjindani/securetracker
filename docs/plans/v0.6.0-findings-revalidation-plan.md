# v0.6.0 Plan: Findings, Evidence, and Revalidation

## Summary

Implement findings lifecycle management after the v0.5.0 Report Repository. Findings move from vendor creation through Paysys assignment, developer remediation evidence, and Apprise revalidation.

## Key Changes

- Add findings APIs for list, create, detail, update, assignment, status change, evidence upload, and revalidation result.
- Add finding status history, evidence, and revalidation data models.
- Add role-based workflow enforcement for Vendor Admin, Paysys Security Admin, and Paysys Developer.
- Add engagement detail UI sections for findings, assignment, evidence, and revalidation.
- Audit finding creation, updates, assignment, status changes, evidence upload, revalidation request, pass, and fail events.

## Test Plan

- Run SDLC gates: `db:generate`, `typecheck`, `lint`, `test`, `build`, and `docker compose config --quiet`.
- Run regression gates: `test:e2e`, `test:regression`, `test:data:cleanup`, and `reset:seeded`.
- Verify full finding flow: vendor create, Paysys assign, developer evidence/fix, Paysys request revalidation, vendor fail, developer refix, vendor pass.

## Assumptions

- Findings are manually entered in v0.6.0; no report parsing or AI extraction is included.
- Evidence may include files in MinIO, but evidence files do not use PDF password handling.
- Risk acceptance, dashboards, notifications, and audit search remain later slices.
