# v0.18.3 Plan: Notifications and Email Alerts

## Summary

Implement in-app notifications and Mailpit-backed email alerts for the current workflow baseline. This slice follows `AGENTS.md` version allocation: after `git fetch origin --prune`, `origin/main` is the only active remote branch and the highest allocated version is `v0.18.2`, so this slice uses `v0.18.3`.

## Key Changes

- Add a notification data model linked to users, workflow entities, read state, email delivery state, and delivery errors.
- Add authenticated notification APIs for list, unread count, mark read, mark all read, and System Admin due-check execution.
- Trigger notifications for engagement initiation/closure, report upload/version upload, finding assignment, revalidation completion, finding due reminders, overdue findings, and risk acceptance expiry.
- Send email through SMTP/Mailpit best-effort; email delivery failure must not block the originating workflow action.
- Add a Notifications frontend page with unread count badge, unread filter, mark-read actions, and System Admin manual due-check action.
- Update cleanup/reset so notification records linked to regression data are removed.

## Test Plan

- `npm.cmd run db:generate`
- `npm.cmd run typecheck`
- `npm.cmd run lint`
- `npm.cmd test`
- `npm.cmd run build`
- `docker compose config --quiet`

## Assumptions

- Mailpit remains the local/dev SMTP target.
- Notification settings UI is reserved for the later administration/configuration slice.
- Due/overdue/expiry checks are manually runnable by System Admin for deterministic local regression.
