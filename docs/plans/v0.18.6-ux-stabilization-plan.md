# v0.18.6 UX Stabilization and Data Quality Plan

## Summary

Implement the first production-readiness stabilization slice after `v0.18.5`. This baseline converts the remaining card-style Audit list to a table, makes Default Page Size a System Admin-managed setting, keeps Calendar month filtering API-backed, and preserves duplicate-create protection for planned VAPT calendar entries.

## Key Changes

- Add `GET /settings` and `PATCH /settings` for portal settings, starting with `DEFAULT_PAGE_SIZE`.
- Store settings in PostgreSQL and audit setting updates with `SYSTEM_SETTING_UPDATED`.
- Add a System Admin Settings page and navigation item.
- Drive frontend table pagination from the backend setting across list-heavy pages.
- Convert Audit search results to a paginated table.
- Extend `GET /calendar` with `startingMonth` server-side filtering.
- Keep duplicate Calendar create behavior idempotent for identical planned entries.

## Test Plan

- Run `db:generate`, `typecheck`, `lint`, `test`, `build`, and `docker compose config --quiet`.
- Verify settings service defaults, allowed page sizes, rejected page sizes, and audit logging.
- Verify Calendar year/month filters and duplicate create behavior.
- Run full regression, cleanup, and seeded reset against the Dockerized stack.

## Assumptions

- `DEFAULT_PAGE_SIZE` allowed values are `10`, `25`, `50`, and `100`.
- Settings are global portal settings for the production-readiness pilot, not per-user preferences.
- Advanced production settings remain for later production-readiness slices.
