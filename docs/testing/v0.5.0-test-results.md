## Coverage Added

- Report upload creates report and first version metadata.
- Report versions store hash, file size, MIME type, MinIO object key, uploader, and protected flag.
- Draft report upload can advance an engagement to `DRAFT_REPORT_UPLOADED`.
- Protected PDF marker detection sets `isPasswordProtected`.
- PDF password fields are rejected by backend upload DTO handling.
- Report view/download actions are audited.
- Read-only roles cannot upload reports.
- Final report versions are blocked after engagement closure.
