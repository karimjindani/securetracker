# v0.18.7 System Configuration User Guide

## Settings

System Admin users can open **Settings** to manage global portal behavior.

Sections:

- **Portal Defaults**: Default Page Size for paginated tables.
- **Schedule Health**: Warning Window Days for On Track / Needs Attention schedule classification.
- **Notifications**: finding reminder days, risk acceptance expiry reminder days, email enablement, and scheduled due checks.
- **Audit and Retention**: audit retention target used for operational planning.

Only `SYSTEM_ADMIN` can save settings. Other authenticated governance users can read settings used by the UI, but cannot update them.

## Notification Scheduler

When **Scheduled due checks enabled** is on, the backend periodically checks whether due reminders, overdue finding alerts, and accepted-risk expiry alerts should be created. Existing daily de-duplication prevents duplicate reminders for the same user/entity/type on the same day.

## Defaults Restored by Reset

`npm.cmd run reset:seeded` restores:

- Default Page Size: `10`
- Warning Window Days: `7`
- Finding Reminder Days: `7`
- Risk Expiry Reminder Days: `14`
- Email Notifications Enabled: `true`
- Scheduled Due Checks Enabled: `false`
- Audit Retention Days: `365`
