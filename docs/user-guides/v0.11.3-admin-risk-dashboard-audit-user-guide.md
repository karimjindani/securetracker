# v0.11.3 Admin, Risk, Dashboard, and Audit User Guide

## Organizations

Open **Organizations** from the left navigation. Users with navigation access can view organization records. `SYSTEM_ADMIN` can create or update the organization name, type, and status.

## Users

Open **Users** as `SYSTEM_ADMIN`. The page shows local user metadata synchronized with Keycloak. Create or update the organization, Keycloak user id, full name, email, role, and status. Other roles do not see the Users navigation item and remain blocked by backend authorization.

## Risk Acceptance

Open an engagement and use the **Findings** section.

- Paysys Security Admin can request risk acceptance for an unresolved finding.
- The request requires risk description, business justification, and expiry date.
- NBP Security Admin can approve or reject requested risk acceptances from the finding card.
- Approved requests move the linked finding to `RISK_ACCEPTED`.

## Dashboard

Open **Dashboard** to view live counts for engagements, findings, overdue work, accepted risks, expiring risks, revalidation success rate, application heatmap, upcoming engagements, and vendor performance.

## Audit

Open **Audit** as System Admin, Auditor, NBP Security Admin, or Paysys Security Admin. Filter by action, entity type, and date range. Select **CSV** to export the filtered records.

## Demo Login Note

The local demo System Admin username remains `system.admin` with the documented development password. A `super.admin` display name is not configured unless the Keycloak seed is changed in a future slice.
