# v0.18.4 Seeded Validation User Guide

## Seeded Data

Run:

```powershell
npm.cmd run reset:seeded
```

This synchronizes the local schema and restores the validation baseline: 3 organizations, 7 demo users, 25 applications, and 50 annual Whitebox engagements.

## Organizations

Organizations represent the three workflow parties:

- NBP: Bank / Client and governance authority.
- Paysys Labs: SaaS service provider, portal operator, and development owner.
- Apprise: VAPT service provider / external security vendor.

The Organizations page shows the friendly workflow party label, portal role, user count, vendor engagement count, and status. The underlying database still stores compact enum values such as `NBP`, `PAYSYS`, and `VENDOR`, but the page avoids exposing those raw codes as business labels.

## Engagements Kanban

Open **Engagements** to view lifecycle columns from Planned through Closed and Go-Live. Use the year, status, and search filters to inspect the seeded annual plan.

## List Pagination

List-heavy pages use the system setting **Default Page Size**, currently set to 10 records per page. Pagination controls appear above the table so users can move through records before reviewing rows. Users may also select 5, 10, 25, or 50 records per page.

The VAPT Calendar supports filters for **Year**, **Starting Month**, and **Status**. Adding a planned VAPT entry is protected against accidental double-submit; repeated identical submissions return the existing planned engagement instead of creating a duplicate.
