# v0.4.0 Engagement and Scoping User Guide

## Engagements

Open the frontend and select `Engagements` from the navigation. The list shows planned and active VAPT engagements, including seeded workflow examples after reset.

Use filters to search by title, application, or status. Open an engagement to view its application, assessment type, vendor, lifecycle status, and scoping records.

## Scoping

Paysys Security Admin can add a scoping record from an engagement detail page.

Required scoping fields:

- Meeting date
- Participants
- Included scope

The first initiation meeting should include Paysys Labs and Apprise / External VAPT Vendor. Bank/NBP attendance is optional. Do not enter passwords in test account summaries.

Scoping records start as `DRAFT`. Paysys Security Admin can finalize them to `FINAL`. Finalized scoping records cannot be edited.

## Lifecycle Actions

Available transition buttons appear only when the current user role can perform that action.

- Paysys Security Admin can initiate planned engagements with Apprise.
- Paysys Security Admin or Vendor Admin can move initiated engagements to Apprise Assessment.
- NBP Security Admin is the only role that can mark an engagement `Closed`.
- Paysys Security Admin can move a closed engagement to `Go-Live`.

NBP initial scope approval is not part of this workflow.

## Seeded Test Records

Use `npm.cmd run reset:seeded` or the external Ops Console at `http://127.0.0.1:3300` to restore seeded v0.4.0 records:

- Planned Whitebox engagement
- Paysys-Apprise initiated engagement with draft scoping
- Apprise Assessment engagement with finalized scoping
- NBP closing meeting engagement
- Closed engagement ready for Go-Live testing
