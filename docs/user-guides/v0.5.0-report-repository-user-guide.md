# v0.5.0 Report Repository User Guide

## Reports on Engagements

Open `Engagements`, select an engagement, and use the `Reports` section.

Users with upload permission can add:

- `DRAFT_REPORT`
- `REVALIDATION_REPORT`
- `FINAL_REPORT`
- `RISK_ACCEPTANCE_DOCUMENT`
- `EVIDENCE_DOCUMENT`
- `ADDENDUM`

The file must be a PDF. Add a title, optional description, and optional upload notes.

## Viewing and Downloading

Each report shows its versions, SHA-256 hash prefix, file size, protected flag, and actions.

- Use the view action to open the browser PDF viewer.
- Use the download action to retrieve the original PDF.

Protected PDFs prompt for a password inside the browser viewer only. Do not enter PDF passwords into upload notes, descriptions, or any backend form field.

## Governance Rules

- Vendor Admin can upload reports for Apprise engagements.
- Paysys Security Admin and System Admin can upload reports for administrative support.
- Draft report upload may move an Apprise assessment engagement to `DRAFT_REPORT_UPLOADED`.
- Final reports cannot be versioned after the engagement is `CLOSED` or `GO_LIVE`.
- Report Repository does not include formal Scope Documents.
