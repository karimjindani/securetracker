# v0.18.3 Notifications User Guide

## Notifications Page

Open **Notifications** from the left navigation. The page shows workflow alerts assigned to your SecureTracker user, including assignment, report, revalidation, due reminder, overdue, risk expiry, engagement initiation, and engagement closure alerts.

Use **Unread only** to focus on new items. Use **Mark read** for one item or **Mark all read** to clear your unread count.

## Email Alerts

Local/dev email is sent through Mailpit. Open the SMTP test UI at `http://localhost:8025` to review delivered notification emails. If SMTP is unavailable, SecureTracker keeps the in-app notification and records the email failure without blocking the workflow action.

## System Admin Due Checks

System Admin users can run **Run checks** on the Notifications page. This evaluates due-soon findings, overdue findings, and expiring risk acceptances using the configured reminder windows.
