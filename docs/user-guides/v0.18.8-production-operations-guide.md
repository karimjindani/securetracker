# v0.18.8 Production Operations Guide

## Docker VM Pilot

Use `docker-compose.prod.yml` for the production pilot baseline. Copy `.env.production.example` to a secure host-managed `.env.production` file and replace every `change-me` value before deployment.

Production mode is enabled with:

```text
SECURETRACKER_DEPLOYMENT_MODE=production
```

When this mode is enabled, the backend fails startup if required database, OIDC, object storage, SMTP, or frontend origin values are missing.

## External Services

- OIDC should point to the NBP/client identity provider. Local Keycloak is for development and CI only.
- SMTP should point to the production mail relay. Mailpit is for local and CI only.
- PostgreSQL and MinIO volumes must be backed up before upgrades.
- The external Ops Console remains a local/dev tool and is not deployed to production.

## Reverse Proxy and TLS

Terminate TLS at the VM reverse proxy, then route:

- `PUBLIC_FRONTEND_URL` to the frontend container on port `5173`.
- `PUBLIC_BACKEND_URL` to the backend API on port `3000`.

Set `CORS_ORIGIN` to the exact frontend HTTPS origin.

## Validation

Before pilot cutover:

```powershell
docker compose -f docker-compose.prod.yml config --quiet
docker compose -f docker-compose.prod.yml build
```

Do not run `npm.cmd run reset:seeded` against a production database.
