# v0.6.0 Findings and Revalidation User Guide

## Findings

Open an engagement and use the `Findings` section.

Vendor Admin, Paysys Security Admin, and System Admin can create findings with reference, title, description, severity, optional CVSS, recommendation, CWE, OWASP category, and due date.

## Assignment and Remediation

Paysys Security Admin assigns findings to Paysys Developers. Assigned developers can:

- Start work by moving a finding to `IN_PROGRESS`.
- Upload remediation evidence with notes, JIRA reference, Git commit, and optional file attachment.
- Mark the finding `FIXED_PENDING_REVALIDATION`.

## Revalidation

Paysys Security Admin requests revalidation after developer fixes. Vendor Admin records revalidation as:

- `PASSED`, which moves the finding to `REVALIDATION_PASSED`.
- `FAILED`, which moves the finding to `REVALIDATION_FAILED` and routes the engagement back to developer fix.

Risk acceptance is not part of v0.6.0.
