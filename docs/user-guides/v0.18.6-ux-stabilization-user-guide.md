# v0.18.6 UX Stabilization User Guide

## Settings

System Admin users can open **Settings** from the left navigation and update **Default Page Size**. The setting controls how many rows appear by default in paginated tables across the portal.

Allowed values are:

- 10 records
- 25 records
- 50 records
- 100 records

Changing the value updates open table pages through the shared portal setting. The backend records a `SYSTEM_SETTING_UPDATED` audit event for each saved change.

## Tables and Filters

Applications, VAPT Calendar, Organizations, Users, Notifications, and Audit use table pagination. Pagination controls appear above the table.

The VAPT Calendar supports:

- Year filter
- Starting Month filter
- Status filter

Calendar creation remains protected against accidental duplicate submission. Submitting the same planned calendar entry again returns the existing engagement instead of creating another row.

## Organizations

Organizations represent workflow parties:

- NBP: Bank/client and governance owner
- Paysys Labs: SaaS service provider and delivery operator
- Apprise: External VAPT service provider
